Fitment Notes - Fitments

Note Fitments Import/Export

Use standard product application fitment import format but add a column 'notes' that has the comma seperated note codes that you created in the previous step (example: "123,abc,xyz")



Note: If you have existing application fitment data, clear them out under schema->manage (before clearing your data make a full backup). This is because if you add a fitment without notes, and later specify notes, the formerly added noteless fitment will take precedence. This will be resolved in a future version of the extension.

Import By Note Message

In the "notes" column there can be a code. If there is a message in the "note_message" column, first the system will check if an existing note has been setup, if so it will use that code, that way messages can be renamed in one place in the admin panel.

Example #3 - Fitment Notes Import (Note By Message)

If that message had never been encountered, it will try to generate a unique code "code-1", "code-2", "code-3" for each new message.

Note Multiple messages - If you need to specify multiple messages you will need to repeat the row, since a message may contain a literal comma. If you need a way to specify multiple messages per fitment, simply copy the row for each message like in example 3.
